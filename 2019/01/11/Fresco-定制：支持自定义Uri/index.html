Fresco 定制：支持自定义Uri

Jan 11, 2019 | Hits

公司最近有一些变动，可以不用去上班。所以多了一些看书和总结的时间。于是想写写博客吧，提炼一下这分工作里做的一些特别的事情。

背景

公司做的是一款及时通信类的社交产品，用户使用的时候，能像微信一样收发图片。但是图片的上产和下载不是基于HTTP实现的，而是基于Socket长连接实现的。通过Socket，移动端上传图片到服务端，如果成功，服务端返回的是一个fileId，大概是这样a1/a00/00/08/rBBkPFjIm3.png。图片的下载，同样走Socket。
服务端一开始这样的设计的初衷，是考虑到用户文件的安全性和隐私性。但是做图片加载的时候，问题来了。

Fresco介绍

Fresco是FaceBook开源的图片加载框架，很强大，有很多好用的feature，支持进度条，淡入淡出等，在低端机也有能有不错的性能。由于对webp的支持，我们选择了Fresco。

问题

Fresco的使用很简单，这里不再累述。使用DraweeController加载图片时，需要设置图片文件明确的Uri。于是，我们打算这样做：构造一个一个Uri字符窜，比如app://a1/a00/00/08/rBBkPFjIm3.png，设置给DraweeController，再将Socket下载图片的功能封装给Fresco用。
撸了一遍官方文档后，发现一个Uri相关的事：Fresco只支持这些Uri，见下图。在好奇心的驱使下，我强行使用了app://xxx，然后图片并没有加载出来，log中出现这么一行Unsupported uri scheme for encoded image fetch! Uri is: XXX

终上，当时我们有两个问题要解决：

  1. 扩展Fresco支持的Uri
  2. 自定义Uri图片下载

二、自定义Uri图片文件下载

这次涉及到fetcher层。fetcher层做是资源文件匹配、获取的操作。
HTTP图片资源使用的是HttpUrlConnectNetworkFetcher,父类是BaseNetworkFetcher。所以我们写一个自己的networkFetcher，叫AppNetworkFetcher。在这个fetcher中，使用socket下载图片文件。
fetcher写好之后，我们要将想办法把它设置给Fresco，这样才可以在匹配app://XXX开头的资源的时候，使用我们自己的networkFetcher。设置方法也很简单，就是在Fresco初始化的时候，调用Fresco#init(Context context, ImagePipelineConfig imagePipelineConfig )方法。ImagePipelineConfig是Fresco的主配置，包括缓存策略、图片解码等等，也包括匹配网络资源的fetcher的指定。
